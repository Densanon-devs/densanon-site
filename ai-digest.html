Research & Papers

Operating-Layer Controls for Onchain Language-Model Agents Under Real Capital

ArXiv cs.AI
  • 21-day live deployment: 3,505 user-funded agents traded real ETH on a bounded onchain environment — one of the first published studies of LLM agents under actual financial stakes.
  • Researchers found that operating-layer controls (validation layers between user mandate and tool execution) are the critical reliability lever, not model capability alone.
  • Agents without operating-layer guardrails failed in correlated ways under adversarial or edge-case market conditions, showing model-level alignment is insufficient for autonomous capital management.
  • Provides a concrete framework for auditing LLM agent reliability before real-world deployment in high-stakes domains.

One Word at a Time: Incremental Completion Decomposition Breaks LLM Safety

ArXiv cs.CL
  • Introduces ICD (Incremental Completion Decomposition): feeds harmful requests one token or phrase at a time across multiple turns, evading turn-level safety filters.
  • The attack works because safety training is typically evaluated on complete prompts — decomposing a request across a trajectory bypasses the classifier's trigger conditions.
  • Effective across multiple frontier models without model-specific tuning or white-box access.
  • Highlights a structural gap in current refusal-training pipelines that evaluate single-turn inputs rather than multi-turn trajectories.

OMEGA: Optimizing Machine Learning by Evaluating Generated Algorithms

ArXiv cs.AI
  • OMEGA is a full end-to-end framework for automated AI research: from idea generation through algorithm implementation to executable benchmark evaluation, no human in the loop.
  • Uses structured meta-prompting to guide LLMs through hypothesis formation, then validates generated algorithms against standard ML benchmarks automatically.
  • Represents a step toward AI systems that can meaningfully accelerate their own research cycle, not just synthesize existing literature.
  • Early results show the system can produce novel algorithmic variants that match or beat baselines on targeted tasks.

Evaluating Strategic Reasoning in Forecasting Agents

ArXiv cs.AI
  • Introduces BTF-2 (Bench to the Future 2): 1,417 pastcasting questions with a frozen 15M-document research corpus, designed to isolate strategic reasoning from factual recall.
  • Standard forecasting leaderboards measure accuracy but reveal nothing about why some agents outperform — BTF-2 adds structured reasoning traces to close that gap.
  • Top agents use qualitatively different search and synthesis strategies, not just better base models.
  • The frozen corpus design prevents data contamination from live web access, making results more reproducible than live-internet benchmarks.

LLM 0.32a0 is a major backwards-compatible refactor

Simon Willison
  • Simon Willison's `llm` CLI/library drops its prompt-and-response model in favor of a first-class conversation and tool-call abstraction — a fundamental rethinking of how the library represents interactions.
  • The refactor is backwards-compatible but changes how sessions are stored and replayed from SQLite, enabling richer multi-turn and agentic workflows.
  • Tool-calling conversations can now be correctly reinflated from persistent storage, fixing a long-standing gap for stateful LLM pipelines built on `llm`.
  • Alpha release (0.32a0/a1) — worth testing now if you use `llm` in scripts or notebooks, but treat as unstable until GA.

AI evals are becoming the new compute bottleneck

Hugging Face Blog
  • Evaluation runs — not training — are now the dominant compute cost for many model development teams, as benchmark suites have grown to thousands of tasks and multi-model comparison grids.
  • The post argues the field needs 'eval evals': meta-evaluations that assess which benchmarks are actually informative vs. redundant, so teams can prune eval suites without losing signal.
  • Practical strategies covered include shared eval caches, lightweight proxy evals for rapid iteration, and separating capability evals from safety evals on different cadences.
  • Directly relevant to anyone running recurring model comparisons — the cost of knowing which model is better is becoming non-trivial.

The Zig project's rationale for their firm anti-AI contribution policy

Simon Willison
  • Zig bans LLM-generated content from all issues, PRs, comments, and bug tracker translations — one of the strictest policies of any major open source project.
  • The rationale centers on review burden: LLM output shifts work from the submitter to the maintainer, who must spend more effort verifying correctness than the contributor saved generating it.
  • The policy also cites signal degradation — LLM-generated issues and PRs dilute the high-quality, precise communication the Zig compiler project depends on.
  • A useful counterpoint to 'AI speeds up open source' narratives; the tradeoffs differ sharply depending on project culture and review bandwidth.

Anthropic GitHub
  • v0.1.70 fixes a silent data loss bug: in-process MCP tool results were lost with older `mcp` library versions, causing the model to receive a validation error instead of actual tool output — floor bumped to `mcp>=1.19.0`.

  • Upgrade immediately if you use in-process MCP tool handlers — the silent failure mode in prior versions could cause hard-to-debug agent behavior.
