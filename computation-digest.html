Linux Kernel's Minimum Rust Version Supported Prepares For Rust 1.85 Baseline

Phoronix
  • The Rust-for-Linux team is preparing to raise the minimum supported Rust version to 1.85, which enables use of more modern language features in kernel drivers and subsystems.
  • Raising the baseline periodically is necessary to let kernel Rust code use stabilized APIs and avoid accumulating technical debt from working around language gaps.
  • The move signals the Rust-in-kernel effort is maturing: version floors only get raised when there is enough downstream adoption confidence to absorb the update requirement.

New Patches Allow Building Linux IPv6-Only, Option To Deprecate 'Legacy' IPv4

Phoronix
  • New kernel patches introduce a build-time option to compile Linux with IPv6 only, allowing operators to shed IPv4 code for embedded, container, or future-network deployments where it is not needed.
  • Marking IPv4 as 'legacy' in kernel terms is a significant philosophical step — it formalizes the idea that IPv4 is no longer the assumed default protocol.
  • For IoT and hyperscale operators already running dual-stack or IPv6-only internally, an IPv6-only kernel build could reduce attack surface and binary size simultaneously.
