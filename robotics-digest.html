Research & Papers

Learning Energy-Efficient Air–Ground Actuation for Hybrid Robots on Stair-Like Terrain

ArXiv cs.RO
  • A new reinforcement learning framework trains a single continuous policy to coordinate propellers, wheels, and tilt servos without predefined mode switches.
  • In simulation, the learned policy achieves ~4x lower energy consumption than propeller-only flight for small obstacle traversal.
  • Hardware transfer to the DoubleBee prototype on an 8cm gap-climbing task showed 38% lower average power versus a rule-based controller — a strong sim-to-real result.

SpatialPoint: Spatial-aware Point Prediction for Embodied Localization

ArXiv cs.RO
  • SpatialPoint formalizes 'embodied localization' — predicting executable 3D points (touchable surfaces and free-space targets) from visual and language inputs.
  • Unlike most vision-language systems that rely on RGB alone, SpatialPoint natively incorporates RGB-D depth data, improving cross-scene generalization.
  • The framework directly addresses a core gap in deploying VLA models on real robots that already carry depth sensors.

Resource-sharing boosts robotic resilience

Robohub

ROS2 Launch File Validation

ROS Discourse
  • A new open-source XSD schema enables real-time XML launch file validation in ROS2, catching syntax errors before runtime.
  • IDE support means common mistakes like using `$(find my_pkg)` instead of `$(find-pkg-share my_pkg)` are flagged immediately during authoring.
  • Command-line validation via xmllint is also supported, making it easy to integrate into CI pipelines.

RFC: Open standard for robot-to-human light signaling — looking for technical feedback from ROS 2 developers

ROS Discourse
  • LSEP (Luminae Signal Expression Protocol) proposes a 9-state machine standard for how robots communicate intent and safety status to nearby humans via LEDs.
  • Currently, every platform invents its own LED semantics, creating confusion in shared workspaces — LSEP aims to solve this interoperability gap.
  • Designed as an isolated ROS 2 safety node, it reads from existing perception pipelines (TTC, proximity) and outputs standardized signals with no platform lock-in.
